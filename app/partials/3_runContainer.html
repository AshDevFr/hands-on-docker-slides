Running a container

RUN Command

$ docker run [OPTIONS] IMAGE[:TAG|@DIGEST] [COMMAND] [ARG...]

Some options:

  • Detached (-d)
  • Foreground (-a -t -i)
  • Name (--name)
  • Ports (-p)
  • Volumes (-v)
  • Detached vs foreground: When starting a Docker container, you must first decide if you want to run the container in the background in a “detached” mode or in the default foreground mode

Run a 'Hello World' 

$ docker run busybox /bin/echo 'Hello world'

RUN Command

Foreground Option

-a=[] : Attach to `STDIN`, `STDOUT` and/or `STDERR`
-t : Allocate a pseudo-tty
-i : Keep STDIN open even if not attached
  • Without -a, Docker will attach all standard streams
  • For interactive processes, you must use -i -t (or -it) together.
  • Specifying -t is forbidden when the client standard output is redirected or piped: 
    $ echo / | docker run -i busybox ls

START command

Start one or more containers 

$ docker start [OPTIONS] CONTAINER [CONTAINER...]

Some options: 

-a, --attach        Attach STDOUT/STDERR and forward signals
         -i, --interactive   Attach container's STDIN

STOP command

Stop a container by sending SIGTERM and then SIGKILL after a grace period 

$ docker stop [OPTIONS] CONTAINER [CONTAINER...]

Some options: 

-t, --time=10      Seconds to wait for stop before killing it

KILL command

Kill a running container using SIGKILL or a specified signal 

$ docker kill [OPTIONS] CONTAINER [CONTAINER...]

Some options: 

-s, --signal="KILL"    Signal to send to the container
