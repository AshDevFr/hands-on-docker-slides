Link

Communication across links

Links allow containers to discover each other and securely transfer information about one container to another container. 

$ docker run --link <name or id>:alias <image>

Docker will defines a set of environment variables for each port exposed by the source container.
Each variable has a unique prefix in the form: 

<name>_PORT_<port>_<protocol>

Setup a link

First image, a database: MongoDB 

$ docker run --name myDb mongo

Second image, your application 

$ docker run --link myDb:db node
